# 쿠키와 세션

<br/>

### 쿠키

- 쿠키는 서버에서 클라이언트한테로 보내지는 작은 데이터다.
- 키/값 구조
- 유효기간 있음(브라우저에서 삭제)
- 서버랑 클라이언트는 기억 못하는 애들임 (무상태성:서버에 부하를 주지 않음)
- 기억시키려고 쿠키를 이용!
- 보안에는 취약해서 httpOnlysecure 같은 게 필요


<br/>
<br/>

### 세션

<br/>

- 쿠키를 보완하기 위한 것
- 민감한 정보는 세션저장소에 보관하고, id 정보만 쿠키가 가지고 다님
- 리소스 비용 절감 but, 서버 측에서는 부담(설계적 측면에서 이해가 필요)
- 결국 중요정보를 서버에 두어 쿠키보다 보안이 좋지만, id가 누적되면 메모리를 많이 차지함.
- 사용자에 대한 정보를 서버에 두기 때문에 쿠키보다 보안에 좋지만, id 늘어날수록 서버 메모리를 많이 차지
- 동접자 수가 많은 웹 사이트는 서버에 과부하를 와서 성능 저하
- 클라이언트가 Request를 보내면, 해당 서버의 엔진이 클라이언트에게 유일한 ID를 부여하는 데 이것이 세션 ID입니다.

# IP, Port 그리고 DNS

## 네트워크

- 두대 이상의 컴퓨터가 연결된 통신망
- 어떻게 데이터가 오가는지가 핵심

## 네트워크의 구성요소

- 호스트(Host): 구성된 네트워크에서 연결된 개별 컴퓨터

- 스위치(switch): 많은 컴퓨터를 한데 묶는 것/ 동일한 네트워크에서 호스트간의 통신을 가능하게 함

다른 네트워크는?!
- 라우터(router): 스위치 기능 + 서로 다른 네트워크 간의 통신 가능하게 함 (공유기)

!!거대 네트워크 = 인터넷!!


<br/>
<hr/>

## IP (Internet Protocol)

- 컴퓨터 간 데이터를 주고받는 네트워크 계층의 규칙

- 데이터 전달에 필요한 목적지 컴퓨터 정보 필요

## IP 주소

- 네트워크에서 컴퓨터가 부여받는 고유한 주소


## 주소 체계 2가지 : IPv4 vs IPv6
- 4하다가 부족해서 6 생김
- ip 는 2진수의 실제 값으로 되어있지만 사용할 떄는 10진수로 변환
- 0~255 사이의 숫자로 이뤄짐

<br/>

## 공인 IP vs 사설 IP

<br/>

### 공인 IP (Public IP)

- ISP가 가정에 공인 아이피를 부여해줌(사실 뻥임)
- 전체 인터넷 망에서 고유하게 식별 가능한 주소
- IPv4 체계에서 자원 부족 (6탄생의 배경)
- 하나의 공인 IP에서 사설 IP 많이 할당해줄 수 있음(효율적)

<br/>

### 사설 IP (Private IP)

- 가정의 LAN과 같은 네트워크에서 할당되는 주소

- 컴퓨터에서 조회되는 IP

<br/>


<br/>

#### 127.0.0.1
 
 - LOCALHOST : 컴퓨터가 스스로에게 붙인 닉
- 자신의 컴퓨터에서만 유효함.

- 내 아이피 찾기 : 네이버에서 IP 주소 조회(방화벽설정가능)

결론 : ISP > 공인 > 사설


<br/>
<br/>

## Port

- 하나의 컴퓨터에 다양한 서비스 실행 중
- 서버도 많은 일하는데 정확한 위치를 가야함(몇호인지)
IP만으로는 충분치 않음
- 서비스를 구분하는 역할
- 미국에서 관리함

<br/>
<br/>

- HTTP 80
- HTTPS 443
- SNTP 25
- ETP 21
=> Protocol과 포트번호
<br/>
<br/>

- 접근하려는 서비스의 목적지 포트를 정확하게 설정
- 안하면 에러

## DNS (Domain Name Server)


- URl을 해석하여 IP 주소로 변환하는 서버
- 전화번호부에 이름치면 번호까지 뜨는 거랑 비슷
- 사용자가 검색하면 DNS서버에서 매칭되는 게 있는지 찾음(전세계에 분포되어있는데 다 뒤짐)
 URL 맵핑된 서버 찾으면 다시 브라우저에 IP주소 전달함
 - 브라우저는 그거 받아서 구글 들어감 
 - 중간다리인 이것이 있다고 알고 있어라 
 - 클라이언트가 서비스를 이용할 수 있는 관문 그래서 이거 망가지면 끝.
