<h1>WEB</h1>


웹(Web)은 인터넷(Internet)을 통해 전 세계적으로 연결된 컴퓨터 네트워크에서 서로 다른 컴퓨터들을 연결하는 시스템입니다. 보통 인터넷을 통해 웹 브라우저를 사용하여 다양한 정보와 자료를 검색하고 열람할 수 있습니다. 웹은 HTML(Hyper Text Markup Language)이라는 언어를 사용하여 문서를 작성하고, 웹 서버와 클라이언트 사이에서 데이터를 주고받는 HTTP(Hyper Text Transfer Protocol) 프로토콜을 사용합니다. 웹은 현재 우리가 인터넷을 사용하는 가장 일반적인 방법 중 하나이며, 검색, 커뮤니케이션, 쇼핑, 업무 등 다양한 용도로 활용됩니다.

<h3>웹의 특징?</h3>

1. 분산성(Distributed): 웹은 전 세계의 수많은 컴퓨터 네트워크와 연결되어 분산된 구조를 가지고 있습니다. 이러한 분산 구조 덕분에 사용자는 언제 어디서든 웹에 접속하여 정보를 검색하고 자료를 열람할 수 있습니다.
2. 열린 표준(Open Standards): 웹은 공개 표준인 HTML, CSS, JavaScript 등의 웹 기술을 사용합니다. 이러한 열린 표준 덕분에 누구나 쉽게 웹 페이지를 작성하고, 웹 서비스를 개발할 수 있습니다.
3. 대화형(Interactive): 웹은 사용자와 상호작용할 수 있는 인터랙티브한 특성을 가지고 있습니다. 사용자는 웹 페이지에서 다양한 요소를 클릭하거나 입력하여 웹 서비스와 상호작용할 수 있습니다.
4. 멀티미디어(Multimedia): 웹은 텍스트, 이미지, 비디오, 오디오 등 다양한 형태의 멀티미디어 자료를 제공합니다. 이러한 멀티미디어 자료는 다양한 웹 페이지에서 사용되며, 사용자에게 더욱 풍부한 경험을 제공합니다.
5. 연결성(Connectivity): 웹은 사용자들 간에 손쉽게 연결될 수 있는 구조를 가지고 있습니다. 이러한 구조 덕분에 사용자는 인터넷을 통해 다른 사용자들과 커뮤니케이션하거나 정보를 공유할 수 있습니다.

<h3>웹의 구성</h3>

웹의 구성은 크게 세 가지 요소로 나눌 수 있습니다: 클라이언트(Client), 서버(Server), 그리고 프로토콜(Protocol)입니다.

1. 클라이언트(Client): 클라이언트는 사용자가 웹 페이지를 보는 데 사용하는 소프트웨어입니다. 대표적으로 웹 브라우저가 있습니다. 웹 브라우저는 사용자가 입력한 URL(Uniform Resource Locator) 주소를 서버에 보내고, 서버에서 받아온 HTML, CSS, JavaScript 등의 문서를 렌더링하여 사용자에게 보여줍니다.
2. 서버(Server): 서버는 사용자가 요청한 정보를 저장하고, 해당 정보를 클라이언트에게 전달하는 역할을 합니다. 서버는 데이터베이스(DB)와 연결되어 사용자가 요청한 데이터를 검색하거나 저장할 수 있습니다. 대표적인 웹 서버 소프트웨어로는 Apache, Nginx, IIS 등이 있습니다.
3. 프로토콜(Protocol): 웹에서 서버와 클라이언트 간에 데이터를 주고받는 데 사용되는 통신 규약입니다. HTTP(Hyper Text Transfer Protocol)는 웹에서 가장 많이 사용되는 프로토콜로, 클라이언트에서 서버로 요청(Request)을 보내고, 서버에서 클라이언트로 응답(Response)을 보내는 방식으로 동작합니다. HTTPS는 HTTP의 보안 버전으로, SSL(Secure Sockets Layer)이나 TLS(Transport Layer Security) 프로토콜을 사용하여 데이터를 암호화합니다.

<h3>웹 서버란?</h3>

웹 서버(Web Server)는 클라이언트가 요청한 웹 페이지나 파일 등의 정보를 제공하는 소프트웨어입니다.

웹 서버는 클라이언트에서 요청(Request)을 받으면, 해당 요청에 대한 응답(Response)을 생성하고 클라이언트에게 전송합니다. 이를 위해서 웹 서버는 HTTP(Hyper Text Transfer Protocol)와 같은 프로토콜을 사용하여 클라이언트와 통신합니다.

웹 서버는 대개 인터넷 상의 도메인 이름(Domain Name)을 기준으로 클라이언트로부터 요청을 받아들입니다. 클라이언트는 도메인 이름을 이용하여 웹 서버에 접속하고, 웹 서버는 해당 도메인 이름에 해당하는 정보를 제공합니다. 이 때, 도메인 이름은 IP(Internet Protocol) 주소로 변환되어 웹 서버에 전달됩니다.

대표적인 웹 서버 소프트웨어로는 Apache, Nginx, IIS 등이 있습니다. 이러한 웹 서버 소프트웨어들은 사용자 요청을 받아들이고, 서버 측 프로그램과 연결되어 동적인 웹 페이지를 생성할 수 있습니다.

<h3>HTTP란?</h3>

HTTP(Hyper Text Transfer Protocol)는 인터넷 상에서 웹 페이지를 전송하기 위해 사용되는 프로토콜입니다.

HTTP는 클라이언트(Client)와 서버(Server) 간에 데이터를 주고받는 데 사용되며, 클라이언트가 서버에게 요청(Request)을 보내면, 서버는 해당 요청에 대한 응답(Response)을 반환합니다.

HTTP는 TCP/IP 프로토콜 위에서 동작하며, 주로 TCP 80번 포트를 사용합니다. 클라이언트는 URL(Uniform Resource Locator)을 이용하여 서버에게 요청을 보내고, 서버는 해당 요청에 대한 응답을 생성하여 클라이언트에게 반환합니다.

HTTP는 단순하고 확장성이 높은 프로토콜로, 웹 브라우저를 비롯한 많은 웹 클라이언트와 서버에서 사용됩니다. 이 외에도, HTTPS(HTTP Secure)와 같이 보안 기능을 추가한 프로토콜도 있습니다.

<h3>HTTP 동작</h3>

HTTP (HyperText Transfer Protocol)는 인터넷 상에서 데이터를 주고받기 위한 프로토콜 중 하나입니다. 웹 브라우저에서 웹 페이지를 요청하면 서버에서는 해당 페이지를 제공하기 위해 HTTP를 사용하여 요청과 응답을 주고받습니다.

HTTP의 동작 방식은 다음과 같습니다.

1. 클라이언트가 서버에 요청(Request)을 보냅니다. 이때 요청은 메서드(GET, POST, PUT, DELETE 등)와 URL, 그리고 필요한 경우 추가적인 데이터(헤더, 바디)를 포함합니다.
2. 서버는 클라이언트 요청에 대한 응답(Response)을 생성합니다. 이때 응답은 상태 코드(200 OK, 404 Not Found 등)와 데이터(헤더, 바디)를 포함합니다.
3. 서버가 생성한 응답을 클라이언트에게 전송합니다.
4. 클라이언트는 서버에서 받은 응답을 처리하고 결과를 화면에 표시합니다.

HTTP는 stateless한 프로토콜이므로, 이전 요청과 현재 요청 간에 어떤 연관성도 가지지 않습니다. 이러한 특성 때문에, 서버와 클라이언트가 특정한 상태를 공유하지 않는 한, 서버는 클라이언트가 이전에 보낸 요청과 현재 보낸 요청이 동일한 것인지 알 수 없습니다. 이를 보완하기 위해, HTTP 쿠키와 세션 등의 메커니즘이 사용됩니다.

<h3>URI란?</h3>

URI (Uniform Resource Identifier)는 인터넷에서 자원을 나타내는 유일한 주소입니다. URI는 URL (Uniform Resource Locator) 및 URN (Uniform Resource Name)을 포함합니다.

URL은 인터넷에서 자원의 위치를 나타내며, 일반적으로 "http://" 또는 "https://"로 시작하는 주소입니다. 예를 들어, "**[https://www.example.com"은](https://www.example.xn--com%22-g040a/)** 인터넷 상의 웹사이트 주소를 나타내는 URL입니다.

URN은 인터넷 상의 자원 이름을 나타냅니다. 예를 들어, "urn:isbn:0-201-61622-X"는 국제 표준 도서번호 (ISBN)를 사용하여 책의 이름을 식별하는 URN입니다.

URI는 URL 및 URN을 모두 포함하며, 인터넷 상의 모든 자원을 식별하는 데 사용됩니다.

<h3>URL이란?</h3>

URL은 "Uniform Resource Locator"의 약어로, 인터넷 상에서 웹 페이지나 파일 등의 위치를 나타내는 주소입니다. 예를 들어, "**[https://www.example.com/index.html"과](https://www.example.com/index.html%22%EA%B3%BC)**
같은 형식으로 작성됩니다. 이 주소는 프로토콜(https), 호스트(**[www.example.com)와](http://www.example.xn--com%29-8j30a/)**
경로(/index.html)로 구성됩니다. URL은 웹 브라우저를 통해 해당 페이지나 파일에 접근할 수 있도록 합니다.

<h3>세션과 쿠키, 토큰</h3>

세션, 쿠키, 그리고 토큰은 모두 웹 애플리케이션에서 인증 및 인가를 위해 사용되는 기술입니다.

세션은 서버 측에서 클라이언트에게 부여되는 일시적인 고유 식별자입니다. 세션 식별자는 클라이언트가 서버에 요청할 때마다 함께 전송되며, 서버에서는 이를 통해 해당 클라이언트가 로그인한 사용자인지 여부를 판단합니다. 일반적으로 세션 식별자는 서버 측에 저장되며, 보안성이 높지만 서버에 부하가 걸리는 단점이 있습니다.

쿠키는 클라이언트 측에 저장되는 작은 데이터 파일입니다. 쿠키를 사용하면 웹 사이트가 클라이언트의 로그인 상태를 유지하고 사용자 정의 기능을 제공할 수 있습니다. 쿠키는 주로 사용자 식별 및 추적, 사용자 설정 저장, 광고 타겟팅 등의 용도로 사용됩니다.

토큰은 인증을 위해 사용되는 문자열입니다. 대개 액세스 토큰(access token)이라고 불리며, 사용자가 로그인하여 인증된 이후에 서버에서 발급됩니다. 액세스 토큰은 보안성이 높고, 서버에 부하가 적으며, 서버와 클라이언트 간의 확장성을 높일 수 있습니다. 보통 JWT(Json Web Token)이나 OAuth2를 이용해 구현합니다.

세션, 쿠키, 그리고 토큰은 모두 인증 및 인가를 위한 중요한 개념입니다. 각각의 장단점과 사용 시나리오에 따라 적절한 기술을 선택하여 웹 애플리케이션의 보안성과 성능을 최적화할 수 있습니다.

<h3>IP</h3>

IP는 "Internet Protocol"의 약어로, 인터넷에서 컴퓨터와 다른 네트워크 장치들이 서로 통신하기 위해 사용되는 프로토콜입니다.

IP 주소는 인터넷 상에서 각 컴퓨터나 네트워크 장치를 식별하는데 사용됩니다. IPv4와 IPv6 두 가지 버전이 있습니다.

IPv4는 32비트 숫자로 이루어진 주소 체계로, 예를 들어 "192.168.0.1"과 같은 형식으로 작성됩니다. IPv4는 주소 고갈 문제와 보안 취약점 등의 이유로 한계가 있어, IPv6가 개발되었습니다.

IPv6는 128비트 숫자로 이루어진 주소 체계로, 예를 들어 "2001:0db8:85a3:0000:0000:8a2e:0370:7334"과 같은 형식으로 작성됩니다. IPv6는 주소 고갈 문제를 해결하고, 더 많은 장치들을 연결할 수 있는 장점이 있습니다.

<h3>PORT</h3>

포트(Port)는 컴퓨터 네트워크에서 프로세스간의 통신 연결을 위해 사용되는 번호입니다.

인터넷에서 데이터를 주고받을 때, 각각의 컴퓨터는 고유한 IP 주소를 갖고 있습니다. 이 IP 주소는 네트워크상에서 해당 컴퓨터를 식별할 수 있는 주소입니다.

하지만, 한 컴퓨터에서 여러 개의 프로세스가 실행될 경우, 이 프로세스들은 각각의 고유한 식별자인 포트 번호를 가져야 합니다. 이 포트 번호는 컴퓨터 내에서 해당 프로세스를 식별하기 위한 것입니다.

예를 들어, 웹 브라우저에서 인터넷에 접속하면, 브라우저는 해당 서버의 IP 주소와 포트 번호를 이용하여 서버와 통신을 합니다. 웹 서버는 일반적으로 80번 포트를 사용하므로, 이 포트 번호를 이용하여 통신을 합니다. 이렇게 포트 번호를 이용하여 프로세스 간에 통신이 이루어지는 것입니다.

<h3>DNS</h3>

DNS는 "Domain Name System"의 약어로, 인터넷에서 도메인 이름을 IP 주소로 변환하는 역할을 하는 시스템입니다.

인터넷에서 컴퓨터나 네트워크 장치들은 IP 주소를 이용해 서로 통신합니다. 하지만, IP 주소는 일반 사용자가 기억하기 어려우므로, 사람이 쉽게 기억할 수 있는 도메인 이름을 사용합니다. 예를 들어, "**[www.example.com"과](http://www.example.xn--com%22-nl6w/)** 같은 도메인 이름은 실제로는 IP 주소로 변환되어 해당 웹 서버에 접속됩니다.

DNS 시스템은 이러한 도메인 이름과 IP 주소를 매핑하여 관리합니다. 일반적으로, 사용자가 도메인 이름을 입력하면, 해당 도메인 이름에 대한 IP 주소를 검색하여 반환합니다. 이때, DNS 서버는 계층 구조로 구성되어 있으며, 최상위 DNS 서버부터 하위 DNS 서버로 이동하며 IP 주소를 검색합니다.

DNS 시스템은 인터넷에서 중요한 역할을 하며, 대부분의 인터넷 사용자들이 이를 이용하고 있습니다.
